# 결정 기록

| 결정                                       | 이유                                                                                            |
| ------------------------------------------ | ----------------------------------------------------------------------------------------------- |
| 바닐라 DOM/CSS + strict TypeScript         | 프롬프트의 기술 제약을 지키고 카드·보드·접근성 DOM을 직접 제어하기 위해서다.                    |
| `src/domain` 순수 함수 분리                | 보드·카드·환승·승리 규칙을 UI와 독립적으로 회귀 검증하기 위해서다.                              |
| 이벤트 위임 한 곳                          | 매 렌더링마다 버튼을 다시 만들어도 listener가 중복되지 않게 하기 위해서다.                      |
| red 공개 모델은 진행 중 `소리 미션`만 표시 | 청음 진행 중 카드 번호·음정명·곡명·반음수를 DOM에 노출하지 않기 위해서다.                       |
| localStorage v2 + Memory fallback          | 개인정보·서버 없이 오프라인 실행하고 quota/보안 예외에서도 앱을 살리기 위해서다.                |
| 수동 service worker                        | 이 저장소는 정적 PWA shell이므로 추가 런타임 의존성 없이 설치·offline 계약을 명시하기 위해서다. |
| 외부 폰트는 폴백 허용                      | 디자인 참고를 반영하되 네트워크가 끊겨도 레이아웃이 유지되어야 하기 때문이다.                   |

## 알려진 수동 게이트

실제 iOS Safari 무음 스위치, 물리 진동, VoiceOver/TalkBack 경험, pinned Linux visual golden은 로컬 자동화만으로 완료를 선언하지 않는다.
